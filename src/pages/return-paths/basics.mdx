<TipBanner>

<div class="mt-6">
  <Card id="why" title="1. 왜 return path 가 중요한가?">
    <div class="prose prose-slate max-w-none">
      <p>
        회로도에서는 신호 화살표가 한 방향으로 가지만, 실제 물리는 <b>닫힌 루프</b> 여야만 전류가 흐릅니다. 
        DRV 가 1 mA 를 내보내면, 같은 1 mA 가 반드시 어딘가를 거쳐 DRV 로 돌아와야 합니다.
      </p>
      <p>
        이 "돌아오는 길 = return path" 는 보통 <b>GND plane</b> 입니다. 고주파일수록 return 전류는 
        "저항이 낮은 경로" 가 아니라 <b>"신호 바로 아래의 자기(mutual) 인덕턴스가 낮은 경로"</b> 를 따라갑니다 — 
        즉, 신호 트레이스 바로 아래로 그림자처럼 붙어서 돌아옵니다.
      </p>
      <p>
        → 결론: <b>루프 면적</b> 이 작을수록 모든 게 좋아집니다. 방사(EMI)도, 잡음 픽업(susceptibility)도, 인덕턴스(L)도 
        모두 루프 면적에 비례합니다.
      </p>
    </div>
  </Card>
</div>

<div class="mt-6">
  <Card id="tight-loop" title="2. 루프 면적이 핵심">
    <ReturnPathLoop />
    <div class="prose prose-slate max-w-none mt-3">
      <p>
        같은 신호라도 return 이 어떤 경로를 타느냐에 따라 <b>인덕턴스가 10 배 이상</b> 차이 납니다. 
        reference plane (보통 인접 층의 GND plane) 이 있으면 return 은 신호 트레이스 바로 아래 유전체 두께(≈ 0.1 mm)만큼의 얇은 loop 을 만듭니다. 
        plane 이 없으면 return 이 수 mm 떨어진 GND 배선까지 우회하고 — 그 순간 고주파 회로는 망가집니다.
      </p>
    </div>
  </Card>
</div>

<div class="mt-6">
  <Card id="slit" title="3. Plane 에 슬릿(slit) 이 있으면 — 재앙">
    <SlitPlaneDisaster />
    <div class="prose prose-slate max-w-none mt-3">
      <p>
        고속 신호가 plane 의 <b>갈라진 부분(slit, split, moat)</b> 을 건너가면, return 전류는 그 갈라진 틈을 
        뚫고 지나갈 수가 없어서 <b>주변을 크게 돌아</b> 갑니다. 이게 가장 흔한 EMI·SI 재난의 원인 중 하나.
      </p>
      <p>
        전형적인 함정들:
      </p>
      <ul>
        <li><b>Analog/Digital GND split</b> — 전도 잡음 분리 목적으로 의도적으로 끊어놨는데 그 위로 고속 신호가 지나감</li>
        <li><b>Anti-pad 줄줄이</b> — BGA 밑에 via clearance 가 일렬로 나열되면 사실상 slit</li>
        <li><b>Power plane 위 라우팅</b> — 신호 아래 plane 이 GND 가 아니라 다른 전원이면, 그 경계가 slit 역할</li>
      </ul>
      <p>
        반드시 건너야 한다면 <b>stitching capacitor (0.1–1 nF MLCC)</b> 를 slit 양쪽 plane 사이에 놓아 return 이 cap 을 통해 점프하도록 합니다.
      </p>
    </div>
  </Card>
</div>

<div class="mt-6">
  <Card id="bounce" title="4. Ground bounce — 공유 return 인덕턴스의 역습">
    <GroundBounce />
    <div class="prose prose-slate max-w-none mt-3">
      <p>
        두 IC 가 긴 배선으로 GND 를 공유하면, 한쪽 IC 의 스위칭 전류가 그 배선 인덕턴스를 지나가며 
        <span class="text-rose-600 font-semibold">V = L · di/dt</span> 의 전압을 만듭니다. 
        다른 IC 입장에서는 <b>자기 GND 기준이 순간적으로 들썩이는</b> 것처럼 보입니다.
      </p>
      <p>
        예시 — 5 nH 의 공유 인덕턴스에 200 mA 가 2 ns 만에 켜지면:
        <br />
        <span class="font-mono">V = 5e-9 × (0.2 / 2e-9) = 0.5 V</span> → 3.3 V 로직이 그냥 <b>흔들립니다</b>.
      </p>
      <p>
        해결 — GND 를 <b>얇은 배선</b> 으로 연결하지 말고 <b>plane</b> 으로, 그리고 각 IC 의 GND 핀은 plane 으로 바로 via drop. 배선 공유를 끊으면 di/dt 가 만드는 bounce 가 공유되지 않습니다.
      </p>
    </div>
  </Card>
</div>

<div class="mt-6">
  <Card id="rules" title="핵심 설계 규칙 (TL;DR)">
    <ul class="list-disc pl-5 space-y-1.5 text-slate-700 [&>li]:leading-relaxed">
      <li>모든 고속 신호 바로 아래·위 층에 <b>solid reference plane</b> (보통 GND) 을 둔다</li>
      <li>신호는 <b>plane 의 slit 을 건너지 않는다</b> — 피해서 라우팅하거나, 불가피하면 stitching cap 으로 점프</li>
      <li>층(layer) 변경 via 는 인접에 <b>GND stitching via</b> 를 둬서 return 이 따라올 수 있게</li>
      <li>GND 는 얇은 트레이스가 아니라 <b>plane</b>. IC GND 핀은 plane 으로 바로 via 를 떨어뜨린다</li>
      <li>민감 회로(ADC, 클록) 와 noisy 회로(스위처, 고속 버스) 의 return 루프가 겹치지 않게 배치</li>
      <li>루프 면적을 "눈으로" 볼 수 있게 의도적으로 라우팅 — 시뮬 없이도 잡히는 50% 의 문제</li>
    </ul>
  </Card>
</div>
