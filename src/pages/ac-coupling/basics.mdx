<div class="mt-6">
  <Card id="why" title="1. 왜 필요한가?">
    <div class="prose prose-slate max-w-none">
      <p>
        두 회로 사이에 신호를 전달할 때, 두 회로의 <b>DC 동작점(bias)</b> 이 다른 경우가 많습니다:
      </p>
      <ul>
        <li>마이크의 출력은 ±수 mV (DC 0 V 기준), 마이크 앰프 입력은 1.65 V 바이어스가 필요 (싱글-서플라이 OP 앰프)</li>
        <li>USB 3 의 TX 쪽은 IC 가 바이어스 결정, RX 쪽은 별도 IC 가 자기 바이어스 가짐 → DC 분리 필요</li>
        <li>고음 분리 — 스피커 크로스오버 — 트위터에는 저주파를 보내지 않기</li>
      </ul>
      <p>
        직류는 차단하고 교류 성분만 통과시키는 가장 단순한 회로 — <b>직렬 캡 + 풀다운/풀업 R</b> = RC 1차 high-pass filter.
      </p>
    </div>
  </Card>
</div>

<div class="mt-6">
  <Card id="hpf" title="2. RC High-Pass Filter — 1차 식">
    <div class="prose prose-slate max-w-none">
      <p>
        신호 → C → R(다음 단 입력 또는 별도 풀다운) → GND. 신호 위치에서 본 임피던스는:
      </p>

      $$|H(f)| = \dfrac{f/f_c}{\sqrt{1 + (f/f_c)^2}}, \quad f_c = \dfrac{1}{2\pi R C}$$

      <ul>
        <li>f ≪ f<sub>c</sub>  → 거의 0 (신호 차단)</li>
        <li>f = f<sub>c</sub>  → −3 dB (약 70%)</li>
        <li>f ≫ f<sub>c</sub>  → 거의 1 (신호 통과)</li>
      </ul>
      <p>
        즉, <b>f<sub>c</sub> 를 신호의 가장 낮은 주파수보다 충분히 낮게</b> 설정하면 됩니다. 보통 10 배 이상 낮게.
      </p>
      <p>
        <b>실 예</b>:
      </p>
      <ul>
        <li>오디오 (20 Hz~20 kHz): f<sub>c</sub> = 2 Hz 정도 → 10 kΩ + 10 μF 또는 100 kΩ + 1 μF</li>
        <li>USB 3 (5 GHz 정도): f<sub>c</sub> = 50 MHz 까지 가능 → 100 nF MLCC + 50 Ω terminator</li>
        <li>마이크 → MCU ADC: f<sub>c</sub> = 10 Hz → 100 kΩ + 100 nF</li>
      </ul>
    </div>
  </Card>
</div>

<div class="mt-6">
  <Card id="droop" title="3. Droop — 사각파에서 보이는 트릭">
    <div class="prose prose-slate max-w-none">
      <p>
        사각파(사인파가 아니라 디지털 펄스) 가 AC 커플링을 통과하면 <b>긴 'high' 또는 'low'</b> 동안 신호가 천천히 0 으로 회귀합니다 — 이를 <b>droop</b> 이라고 합니다.
      </p>
      <p>
        일반 식: T 시간이 지나면 step 의 amplitude 가 <span class="font-mono">exp(−T/τ)</span> 만큼만 남음 (τ = R·C).
        예) τ = 10 ms, T = 1 ms 면 droop = 1 − exp(−0.1) ≈ 9.5 %.
      </p>
      <p>
        <b>USB 3 / SerDes</b> 같은 8b/10b 인코딩은 DC 균형이 보장되어 long run 이 짧아 droop 가 작지만, 
        <b>UART / SPI</b> 처럼 오랫동안 같은 값이 유지되는 신호는 AC 커플링이 부적합합니다.
      </p>
    </div>
  </Card>
</div>

<div class="mt-6">
  <Card id="bypass" title="4. 바이패스 캡 (Bypass) — 같은 회로의 다른 이름">
    <div class="prose prose-slate max-w-none">
      <p>
        IC 의 전원 핀 옆 100 nF 캡(<a href="/pcb/decoupling-capacitor/basics" class="text-indigo-600 underline decoration-indigo-200 hover:decoration-indigo-600">디커플링 캡</a>) 도 같은 원리:
      </p>
      <ul>
        <li>고주파 노이즈 → 캡을 통해 GND 로 빠져나감 (high-pass 의 "차단" 측에서 본 시각)</li>
        <li>DC 전원 → 캡으로는 안 흐름 → IC 핀까지 전달</li>
      </ul>
      <p>
        <b>커플링</b> (직렬) 과 <b>바이패스</b> (병렬) 는 위치만 다를 뿐 <b>같은 동작 (f-dependent Z)</b> 입니다. 
        하나는 신호 라인에서 DC 를 끊는 용도, 하나는 전원 라인에서 AC 를 GND 로 빼는 용도.
      </p>
    </div>
  </Card>
</div>
