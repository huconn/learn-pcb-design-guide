        <li><b>Intra-pair</b>: 길이 차이 &lt; 5 mil (0.13 mm)</li>
        <li><b>AC coupling</b>: TX 측에 100 nF MLCC 직렬 (커넥터 가까이) — RX 측은 호스트가 처리</li>
        <li><b>via stub</b>: 5 Gbps 까지는 일반 via OK, 10 Gbps 부터 <b>back-drill</b> 권장</li>
        <li><b>최대 채널 손실</b>: -7.5 ~ -10 dB @ Nyquist (스펙 따라)</li>
        <li><b>레퍼런스 plane</b>: 절대 끊기지 않게. 페어 아래 GND 가 연속</li>
      </ul>
      <p><b>레이아웃 팁</b></p>
      <ul>
        <li>TX 와 RX 페어는 별도 채널이라 페어끼리 길이 매칭 불필요. 단, 하나의 페어 안 (TX 안, RX 안) 은 매우 엄격.</li>
        <li>AC coupling cap 위치에서 임피던스가 살짝 흔들리는 게 정상. cap 풋프린트는 길이를 짧게(0402 또는 0201).</li>
        <li>코너 라우팅 시 45° 두 번 (mitred) 또는 둥근 호 — 90° 직각 금지.</li>
      </ul>
      <p><b>관련 링크</b> — <a href="/pcb/impedance-matching/basics#reflection">반사 계수</a> · <a href="/pcb/return-paths/basics">return paths</a></p>

    <figure class="not-prose rounded-lg border border-slate-200 overflow-hidden">

    <div class="prose prose-slate max-w-none mt-3">
      <p>
        차동 페어의 두 트레이스가 도착할 때 시간 차이가 나면 (skew), 신호가 깨집니다. 페어 안 길이 매칭(intra-pair) 은 보통 <b>≤ 5 mil</b> 까지. 라우팅 중 한쪽이 더 길어지면 짧은 쪽 (<b>위 그림에서는 TX+</b>) 에 사형 (serpentine) 을 넣어 길이를 늘립니다.
      </p>
      <p><b>요점</b>:</p>
      <ul>
        <li><b>중간에서 처리</b> — 끝단(드라이버 핀 직후 또는 RX 핀 직전) 보다 중간이 임피던스 변동에 강함</li>
        <li><b>짧은 쪽만</b> 보정 — 둘 다 사형으로 만들지 말 것</li>
        <li>사형 폭은 <b>3W 이상</b> — 사형 자체가 자기와 결합해 임피던스를 흔드는 걸 막기 위해</li>
        <li>같은 페어가 via 를 건너면 via 도 같은 위치, 같은 개수로 (페어가 항상 대칭)</li>
      </ul>
      <p>USB 3 / DDR / HDMI / PCIe 모두 같은 패턴.</p>
    </div>
  </Card>
</div>

<div class="mt-6">
  <Card id="gbe" title="Gigabit Ethernet (1000BASE-T)" subtitle="4 페어 · 마그네틱 분리 · Bob Smith 종단">
    <div class="prose prose-slate max-w-none">
      <p><b>핵심 숫자</b></p>
      <ul>
        <li><b>차동 임피던스</b>: 100 Ω ± 10 % (4 페어 모두)</li>
        <li><b>Intra-pair</b>: 길이 차이 &lt; 5 mil</li>
        <li><b>Inter-pair</b>: 페어 간 길이 차이 &lt; 50 mil (양방향 동시 통신이라 페어 간 skew 도 중요)</li>
        <li><b>마그네틱</b>: PHY 와 RJ-45 사이에 반드시 절연 트랜스포머 (전기 분리 + EMI 차단)</li>
        <li><b>Bob Smith 종단</b>: 미사용 페어를 75 Ω + 1 nF 으로 chassis GND 에 연결 → 공통모드 EMI 흡수</li>
      </ul>
      <p><b>구성 순서 (필수)</b></p>
      <ol>
        <li>PHY 칩</li>
        <li>차동 페어 (4쌍)</li>
        <li>마그네틱 트랜스포머 (분리된 모듈 또는 RJ-45 내장형)</li>
        <li>RJ-45 커넥터 (chassis GND 와 연결)</li>
      </ol>
      <p><b>흔한 실수</b></p>
      <ul>
        <li>마그네틱 <b>2차측 GND</b> 를 디지털 GND 와 직접 연결 → 격리 의미 사라짐. chassis GND 만 연결.</li>
        <li>4 페어 라우팅 시 인접 페어 간 간격 부족 → crosstalk → 짧은 cable 에서는 OK 인데 100 m 케이블에서 fail.</li>
        <li>Bob Smith 의 75 Ω 가 정확하지 않거나 cap 이 빠짐 → EMI 시험에서 떨어짐.</li>
      </ul>
    </div>
  </Card>
</div>

<div class="mt-6">
  <Card id="hdmi" title="HDMI (TMDS)" subtitle="3 데이터 + 1 클럭 페어 · DC-coupled">
    <div class="prose prose-slate max-w-none">
      <p><b>핵심 숫자</b></p>
      <ul>
        <li><b>차동 임피던스</b>: 100 Ω ± 10 %</li>
        <li><b>페어 수</b>: TMDS Data 0/1/2 (3) + TMDS Clock (1)  = 총 4 페어</li>
        <li><b>Intra-pair</b>: &lt; 5 mil</li>
        <li><b>Inter-pair (TMDS 4쌍)</b>: 길이 차이 &lt; 200 mil (skew 보정이 한계가 있음)</li>
        <li><b>AC coupling</b>: 없음 — TMDS 는 DC-coupled (송신측 termination 이 DC bias 결정)</li>
        <li><b>ESD</b>: 커넥터 바로 옆에 4-channel ESD array (예: TPD4S009) — 핀 캐패시턴스 1 pF 미만 제품 사용</li>
      </ul>
      <p><b>레이아웃 팁</b></p>
      <ul>
        <li>HDMI 2.0 부터는 6 Gbps/lane 이라 USB 3 와 비슷한 SI 주의가 필요. 가능하면 inner layer 라우팅.</li>
        <li>HPD/CEC/DDC 같은 저속 신호는 별도 영역. TMDS 페어 옆에 두지 않기.</li>
        <li>5V power (HDMI 의 +5V 핀) 는 트레이스가 길면 ferrite bead + bulk cap 으로 노이즈 분리.</li>
      </ul>
    </div>
  </Card>
</div>
