<div class="mt-6">

    <PierceOscillator />
    <div class="prose prose-slate max-w-none mt-3">
      <p>
        대부분의 MCU 는 내부에 인버터(증폭기)와 자동 바이어스 회로를 가지고 있어, 외부에는 <b>crystal + C1 + C2</b> 만 추가하면 발진합니다 — 이를 <b>Pierce 회로</b> 라고 합니다.
      </p>
      <ul>
        <li><b>Crystal (Y1)</b> — 실리콘 양 끝에 전극이 붙어 진동하는 quartz 결정. 정확한 주파수 결정.</li>
        <li><b>C1, C2</b> — 발진을 위한 위상 조건을 만들고, crystal 의 load capacitance C_L 을 매칭.</li>
        <li><b>MCU 내부 R_f</b> — 자동 바이어스. 일부 MCU 는 외부에 추가 저항을 요구.</li>
      </ul>
    </div>
  </Card>
</div>

<div class="mt-6">
  <Card id="loadcap" title="2. Load capacitance — 가장 중요한 식">
    <div class="prose prose-slate max-w-none">
      <p>Crystal 데이터시트의 <b>C_L (load capacitance)</b> — 이 값에 맞춰 발진해야 정격 주파수가 나옵니다. 회로의 두 cap 이 만드는 직렬 합성 + 트레이스/MCU 의 stray cap 이 함께 C_L 을 만듭니다:</p>
      <p class="font-mono text-center text-slate-800 bg-slate-50 border border-slate-200 rounded px-3 py-2 text-base">
        C_L = (C1 · C2) / (C1 + C2) + C_stray
      </p>
      <p>대칭으로 (C1 = C2) 두면 풀이가 간단합니다:</p>
      <p class="font-mono text-center text-slate-800 bg-slate-50 border border-slate-200 rounded px-3 py-2">
        C1 = C2 = 2 · (C_L − C_stray)
      </p>
      <p>
        예) C_L = 18 pF, C_stray = 5 pF → C1 = C2 = 26 pF → 표준 값 22 또는 27 pF 로 round.
      </p>
      <p>
        <b>C_stray 추정</b> — 보통 3 – 7 pF. MCU 데이터시트의 XIN/XOUT pin capacitance (~3 pF) + 짧은 PCB 트레이스 (~2 pF) 합. 모르면 5 pF 로 시작.
      </p>
    </div>
  </Card>
</div>

        <li>Crystal/cap 영역 주변에 <b>GND 가드 링</b> (closed loop) 을 그리고, <b>3 mm 간격으로 stitching via</b> 로 GND plane 까지 연결</li>

        <li>C1, C2 의 GND 측은 <b>별도 via 로 plane 직결</b> — 트레이스로 둘러서 가지 않기</li>
      </ul>
      <p><b>하지 말 것</b>:</p>
      <ul>
        <li>고속 신호 (USB, SPI, PCIe, 클럭 분배) 가 crystal 영역 위/옆을 지나가게 두지 말 것</li>
        <li>XIN/XOUT 트레이스를 다른 신호와 평행으로 길게 두지 말 것</li>
        <li>Via 로 층을 옮겨가며 라우팅하지 말 것 (XIN/XOUT 은 한 층에서 짧게)</li>
        <li>가드 링이 끊어진 채로 두지 말 것 (open guard = 무용지물)</li>
      </ul>
    </div>

    <div class="prose prose-slate max-w-none">
      <ol>
        <li><b>C1/C2 값이 너무 큼</b> — gain margin 부족으로 시동 실패. 30% 줄여보기.</li>
        <li><b>C_stray 과소평가</b> — 트레이스가 길거나 via 가 있으면 stray 가 10 pF 넘을 수 있음.</li>
        <li><b>Drive level 과다</b> — 일부 MCU 는 drive level 설정이 있음 (low/med/high). 작은 crystal 에 high 쓰면 손상.</li>
        <li><b>Crystal 본체가 GND 와 닿음</b> — crystal 케이스가 SMD 바닥과 short. 캐이스용 별도 패드를 GND 로 연결하되 신호 트레이스와 분리.</li>
        <li><b>옆 신호가 crystal 위로 라우팅</b> — jitter / 시동 실패 / 주파수 흔들림.</li>
      </ol>
